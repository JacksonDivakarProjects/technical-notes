If you get a question you don't know, use these points to fill the space. They are generic enough to be correct but specific enough to get marks.

The primary objective of implementing [Insert Topic Name] is to minimize human error and maximize operational throughput."_

---

### **2. The "3-D Rule" (For any Application/Need Question)**

If the question asks **"Why use robots?"**, **"Applications,"** or **"Need for Automation,"** always list the **3 Ds**:

- **Dull:** Robots perform repetitive tasks (like pick-and-place) without fatigue or boredom, ensuring consistent quality.
    
- **Dirty:** Robots can work in environments with oil, grease, paint fumes, or dust that would be uncomfortable for humans.
    
- **Dangerous:** Robots handle hazardous tasks (high heat, radiation, toxic chemicals, or heavy lifting) that pose risks to human health.
    

---

### **3. The "Universal Selection Criteria" (For any Hardware Question)**

If the question is **"Selection of [Sensors / Grippers / Motors / Robots],"** write these 6 bullet points. They apply to _everything_.

1. **Payload/Capacity:** The device must be able to handle the required weight or force (e.g., lifting capacity).
    
2. **Accuracy & Resolution:** The ability to perform tasks with high precision and detect small changes.
    
3. **Speed/Response Time:** The device must react fast enough to keep up with the production line cycle time.
    
4. **Environmental Compatibility:** Must withstand harsh conditions like high temperature, humidity, vibration, and dust (IP Rating).
    
5. **Cost:** Low initial investment and low maintenance cost (Total Cost of Ownership).
    
6. **Interfacing:** Ease of connection with the existing Robot Controller (PLC compatibility).
    

---

### **4. The "Universal Block Diagram" (Draw this if you have no idea)**

This diagram fits almost any question about **Control Systems, Motors, Sensors, or Robot Architecture**.

Draw this structure:

Controller (Brain) $\leftrightarrow$ Driver/Amplifier $\rightarrow$ Actuator (Motor) $\rightarrow$ Process/Load

$\uparrow$ --------------------------------- Sensor (Feedback) -------------------------------------- $\downarrow$

- **Label it:** "Closed-Loop Feedback System."
    
- **Explanation:** "The system continuously monitors the output using sensors and compares it with the desired input to correct errors."
    

---

### **5. The "Universal Advantages" (Pros)**

Write these for any device (Hydraulic, Pneumatic, Sensor, Robot):

- **High Reliability:** Designed for continuous 24/7 operation.
    
- **Improved Quality:** Reduces variations caused by human factors.
    
- **Data Integration:** Can send real-time data to the central server for analysis.
    
- **Scalability:** Can be easily upgraded or reprogrammed for different tasks.
    

---

### **6. The "Universal Disadvantages" (Cons)**

Write these if asked for limitations:

- **High Initial Cost:** Requires significant capital for hardware and installation.
    
- **Skilled Labor:** Requires trained engineers for programming and maintenance.
    
- **Complexity:** Integration with existing legacy systems can be difficult.
    
- **Safety Risk:** Malfunction can cause damage to the product or injury to nearby equipment.
