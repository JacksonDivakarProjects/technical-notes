### **Q3: What is an Expert System? Explain its Architecture and Application in Agriculture.**

#### **1. Introduction (2 Marks)**

- **Definition:** An Expert System (ES) is a computer program that simulates the judgment and behavior of a human or an organization that has expert knowledge and experience in a particular field.
    
- **Goal:** To transfer knowledge from a human expert (e.g., a top agronomist or plant pathologist) to a computer system, so that ordinary farmers can access this expertise anytime, anywhere.
    
- **Difference from Standard Software:** Standard software processes data (calculations); an Expert System processes **knowledge** (rules and logic).
    

#### **2. Architecture of an Expert System (The Core Diagram) (6 Marks)**

An Expert System consists of three main components. You **must** draw a block diagram similar to the one below to get full marks.

**A. Knowledge Base (The "Brain's Library")**

- This is the storage unit where the expert’s knowledge is kept.
    
- It contains specific facts and rules (e.g., "IF leaves have brown spots AND it rained yesterday, THEN probability of Fungal Infection is High").
    
- This knowledge is gathered from human experts and research papers.
    

**B. Inference Engine (The "Reasoning Brain")**

- This is the processing unit. It does not store knowledge; it _uses_ the knowledge.
    
- It applies logical rules to the specific data provided by the user to deduce a conclusion.
    
- It acts like a detective, matching the farmer's symptoms with the rules in the Knowledge Base to find the cause.
    

**C. User Interface (The "Conversation")**

- This is the bridge between the user (farmer) and the system.
    
- It allows the user to query the system (e.g., upload a photo or answer questions) and displays the final advice or diagnosis in simple language.
    

#### **3. Working Mechanism (Step-by-Step) (3 Marks)**

1. **User Input:** The farmer notices yellow leaves and uses the app (User Interface) to answer questions: "Is the soil dry?", "Are there insects?"
    
2. **Processing:** The **Inference Engine** takes these answers and searches the **Knowledge Base**.
    
3. **Matching:** It finds a rule that matches the symptoms (e.g., "Yellow leaves + Dry soil = Nitrogen Deficiency").
    
4. **Output:** The system advises the farmer: "Apply Nitrogen fertilizer."
    

#### **4. Applications in Agriculture (2 Marks)**

- **Crop Protection:** Diagnosing pests and diseases (e.g., an app identifying a specific beetle eating the crop).
    
- **Irrigation Scheduling:** Deciding when to water crops based on soil type and weather rules.
    
- **Farm Management:** Advising on which variety of seed to plant based on the farm's location and soil history.
    
- **Example:** **PROSPECTOR** was an early expert system used in geology, but modern apps like **Plantix** are agricultural examples where the phone camera acts as the input.
